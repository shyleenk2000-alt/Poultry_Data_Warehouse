Poultry_Data_Warehouse
 Project Overview

This project demonstrates the end-to-end process of data modeling, database creation, and data loading using SQL.

The goal is to design a simple data warehouse for a poultry business that tracks sales across customers, products, suppliers, dates, and locations.

---

 Data Modeling Approach

We use a Star Schema, a common structure in data warehousing.

 Key Concepts

- Data Modeling → Designing structure of data
- Star Schema → Fact table + dimension tables
- Primary Key (PK) → Unique identifier
- Foreign Key (FK) → Links between tables
- Fact Table → Stores measurable data
- Dimension Tables → Provide descriptive attributes

Improvements & Future Work

- Calculate "TotalAmount" dynamically instead of storing it
- Add more transactional data for analysis
- Build dashboards using tools like Power BI or Tableau
- Implement indexing for better performance

---

 Conclusion

This project provides a foundational understanding of:

- Designing a data warehouse
- Implementing it in SQL
- Loading and querying structured data

It is a great starting point for learning data analytics, business intelligence, and database design.
